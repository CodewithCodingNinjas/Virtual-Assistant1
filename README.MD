# Virtual Voice Assistant

A bilingual (Hindi/English) voice assistant built on the MERN stack with a
**hybrid intent-recognition engine**: an on-device machine-learning classifier
resolves common commands in under a millisecond with no network call, and a
Gemini LLM is used only as a fallback for open-ended queries.

> **Design highlight:** most commands are understood **offline** by a Naive Bayes
> intent classifier I trained on a labeled bilingual corpus. The paid LLM is
> invoked only when the offline model is not confident — cutting latency, cost,
> and data exposure. See [`docs/ARCHITECTURE.md`](docs/ARCHITECTURE.md).

---

## Key features
- Real-time voice recognition (Web Speech API) and Hindi/English speech synthesis.
- **Offline ML intent classifier** — 14 intents, ~83% cross-validated accuracy,
  sub-millisecond inference, zero runtime dependencies.
- **Hybrid pipeline** — offline-first, LLM fallback, server-authoritative
  date/time. Every response is tagged `source: offline | llm`.
- Quick actions: open/search Google & YouTube, play media, calculator, Instagram,
  Facebook, weather, date/time/day/month.
- JWT (httpOnly cookie) auth with bcrypt password hashing.
- Customizable assistant name & avatar (Cloudinary), persistent history (MongoDB).
- Hardened uploads, per-user rate limiting, validated LLM output, CI with tests.

## The ML intent classifier

Rather than wrapping an LLM for everything, the assistant owns its primary
intent-recognition model.

- **Model:** Multinomial Naive Bayes with Laplace smoothing.
- **Features:** word unigrams + bigrams **and** character 3,4-grams — the
  character features make it robust to romanized-Hindi morphology
  (*kholo / khol / kholna*) and ASR spelling drift on a small corpus.
- **Data:** 238 labeled, bilingual utterances across 14 intents
  (`Backend/ml/intents.dataset.json`).
- **Evaluation:** 5-fold stratified cross-validation, fixed seed.

### Measured results (reproduce with `npm run ml:eval`)

| Metric | Value |
|---|---|
| Cross-validated accuracy | **82.8%** (per-fold std 5.4%) |
| Macro F1 | 82.8% |
| Macro precision / recall | 83.8% / 83.9% |
| Median inference latency | ~0.07–0.25 ms |

Full per-class table and confusion matrix: [`docs/ML_METRICS.md`](docs/ML_METRICS.md)
(auto-generated by `node ml/report.js`).

## Getting started

### Prerequisites
- Node.js v18+ (developed on v22)
- MongoDB (local or Atlas)
- Cloudinary account (avatar uploads)
- Gemini API endpoint/key (LLM fallback)

### Backend
```bash
cd Backend
npm install
cp .env.example .env      # then fill in real values
npm run dev               # or: npm start
```

`.env` keys (see `.env.example` — names must match the code):
```
PORT, NODE_ENV, ALLOWED_ORIGINS
MONGODB_URI            # MONGODB_URL also accepted
JWT_SECRET
CLOUDINARY_CLOUD_NAME, CLOUDINARY_CLOUD_KEY, CLOUDINARY_CLOUD_SECRET
GEMINI_API_URL
```

### Frontend
```bash
cd Frontend
npm install
# set VITE_API_URL in .env (defaults to http://localhost:5000)
npm run dev
```

### Tests & ML evaluation
```bash
cd Backend
npm test          # unit tests (node:test)
npm run ml:eval   # 5-fold cross-validation + latency benchmark
node ml/report.js # regenerate docs/ML_METRICS.md
```
